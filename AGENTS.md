# AGENTS.md - Coding Agent Instructions

## Project Overview

**Monorepo structure** with two main components:
- `api/` — .NET 10 FastEndpoints backend (see `api/src/online/`)
- `client/` — SvelteKit 5 frontend with TypeScript (see `client/src/`)
- **Purpose:** Public booking and member login system with Google OAuth authentication

## Build, Test & Lint Commands

### Backend (API)
```bash
# Run API server (or use VS Code task "API")
dotnet run --project api/src/online/online.csproj --no-restore

# Build
dotnet build api/src/online/online.csproj

# Test
dotnet test api/tests/

# Entity Framework migrations
dotnet ef migrations add <MigrationName> --project api/src/online/online.csproj --context AppDbContext -o Data/Migrations
dotnet ef migrations remove --project api/src/online/online.csproj --context AppDbContext
dotnet ef database update --project api/src/online/online.csproj --context AppDbContext
```

**VS Code Tasks:**
- `API` - Run the backend server
- `EF Migration Add` - Add new migration (prompts for name)
- `EF Migration Remove` - Remove last migration
- `EF Revert` - Revert database to a previous migration (prompts for name)

### Frontend (Client)
```bash
cd client/

# Development
pnpm run dev                    # Start dev server
pnpm run build                  # Production build
pnpm run preview                # Preview production build

# Testing
pnpm run test                   # Run all tests (unit + e2e)
pnpm run test:unit              # Vitest unit tests only
pnpm run test:unit -- --run     # Run unit tests once (CI mode)
pnpm run test:unit -- --watch   # Watch mode
pnpm run test:e2e               # Playwright E2E tests
pnpm run test:install           # Install Playwright browsers

# Run single test file
pnpm run test:unit -- src/lib/components/Button.svelte.test.ts
pnpm run test:e2e -- --grep "login flow"

# Linting & Formatting
pnpm run lint                   # Check prettier + eslint
pnpm run lint:fix               # Fix eslint issues
pnpm run format                 # Format code with prettier
pnpm run check                  # Svelte type checking

# API client generation
pnpm run api                    # Generate TypeScript API client from OpenAPI
```

**API Client Generation:**
- Clients are generated by Orval from `swagger.json` (see `client/orval.config.ts`)
- Custom fetch/mutator logic lives at `client/src/lib/api/mutator/customInstance.svelte.ts`

## Code Style Guidelines

### General Formatting
- **Tabs for indentation** (frontend) / **4 spaces** (backend)
- **Line length:** 100 characters
- **Trailing commas:** ES5 style
- **Quotes:** Double quotes preferred
- **File naming:** kebab-case for files, PascalCase for classes/components

### Frontend (TypeScript/Svelte)

#### Svelte 5 Runes (Required)
```typescript
// State variables - $state() is globally available
let count = $state(0);
let user = $state<User | null>(null);

// Derived state
let doubled = $derived(count * 2);
let complexData = $derived.by(() => {
	// complex logic here
	return processedResult;
});

// Effects (use sparingly, prefer $derived)
$effect(() => {
	console.log(`Count is now ${count}`);
});

// Component props - new syntax
type Props = {
	title: string;
	optional?: boolean;
};
let { title, optional = false } = $props();
```

#### Event Handling
```typescript
// Modern event handling (not onclick|preventDefault)
<button onclick={(event) => {
	event.preventDefault();
	handleClick();
}}>Click me</button>

// Instead of legacy svelte:component
<MyComponent />  // Dynamic by default
<props.icon />   // Or as attributes
```

#### Imports & Exports
```typescript
// Use const for functions instead of function expressions
export const fetchUserData = async (id: string) => {
	return await api.users.getById(id);
};

// Import organization
import type { ComponentType } from 'svelte';

```

#### FastEndpoints Structure
```csharp
// Endpoint naming: {Feature}{Action}Endpoint
public class GetUsersEndpoint : EndpointWithoutRequest<List<UserDto>>
{
	public override void Configure()
	{
		Get("/api/users");
		AllowAnonymous();
	}

	public override async Task HandleAsync(CancellationToken ct)
	{
		// Implementation
		await SendOkAsync(result, ct);
	}
}
```

#### Service Registration
```csharp
// In Common/Extensions/ServiceCollectionExtensions.cs
public static IServiceCollection ConfigureMyFeature(this IServiceCollection services)
{
	services.AddScoped<IMyService, MyService>();
	return services;
}

// Register in Program.cs
builder.Services.ConfigureMyFeature();
```

### Testing Conventions

#### Unit Tests (Vitest)
```typescript
// File naming: ComponentName.svelte.test.ts
// Location: Same directory as component

import { describe, it, expect } from 'vitest';
import { render } from 'vitest-browser-svelte';
import Button from './Button.svelte';

describe('Button', () => {
	it('should render with text', async () => {
		const { getByRole } = render(Button, { props: { children: 'Click me' } });
		expect(getByRole('button')).toHaveTextContent('Click me');
	});
});
```

#### E2E Tests (Playwright)
```typescript
// File naming: feature-name.spec.ts
// Location: client/e2e/

import { test, expect } from '@playwright/test';

test.describe('User Authentication', () => {
	test('should login successfully', async ({ page }) => {
		await page.goto('/');
		await page.click('[data-testid="login-button"]');
		await expect(page).toHaveURL('/dashboard');
	});
});
```

#### Backend Tests
```csharp
// File naming: ClassNameTests.cs
// Use xUnit, arrange-act-assert pattern

[Fact]
public async Task GetUser_ShouldReturnUser_WhenUserExists()
{
	// Arrange
	var userId = Guid.NewGuid();
	
	// Act
	var result = await userService.GetUserAsync(userId);
	
	// Assert
	result.Should().NotBeNull();
	result.Id.Should().Be(userId);
}
```

## Key Development Workflows

### Adding New API Endpoint
1. Create endpoint in `api/src/online/Features/{FeatureName}/`
2. Register services in `Common/Extensions/`
3. Regenerate API client: `pnpm run api` (in client/)
4. Update frontend to use new endpoint

### Adding New Page/Component
1. Create Svelte file in `client/src/routes/` or `client/src/lib/components/`
2. Use Svelte 5 runes syntax
3. Import generated API clients from `$lib/api/generated`
4. Add tests in same directory

### Database Changes
1. Modify entities in `api/src/online/Entities/`
2. Add migration: `dotnet ef migrations add MigrationName`
3. Update database: `dotnet ef database update`

## Important Notes

- **Never commit secrets** - use `dotnet user-secrets` for local development
- **API-first development** - OpenAPI spec drives client generation
- **File-scoped namespaces** required in C# code
- **Use tabs** for frontend indentation, **4 spaces** for backend
- **Svelte 5 runes** are mandatory - avoid legacy Svelte patterns
- **Feature-based organization** for backend code
- **Generated API clients** - don't write manual HTTP calls
